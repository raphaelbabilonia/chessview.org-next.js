# ChessView.org Next.js

Server-rendered public website for ChessView.org.

This app is the SEO-oriented public surface for ChessView. It consumes the existing Express API and renders public event pages as HTML with metadata, sitemap, robots rules, and JSON-LD event data.

## Why Next.js

The React/Vite frontend is a client-side app. That is fine for dashboards and authenticated workflows, but public pages benefit from server-rendered HTML so search engines and social crawlers can read event content immediately.

Initial public routes:

- `/`
- `/en`, `/es`, `/it`
- `/en/events`, `/es/events`, `/it/events`
- `/en/events/[id-or-slug]`, `/es/events/[id-or-slug]`, `/it/events/[id-or-slug]`
- `/en/countries/[country]`, `/es/countries/[country]`, `/it/countries/[country]`
- `/en/sources/[source]`, `/es/sources/[source]`, `/it/sources/[source]`
- `/sitemap.xml`
- `/robots.txt`

## Languages

The public site is localized in:

- English (`/en`)
- Spanish (`/es`)
- Italian (`/it`)

Localized URLs are used for SEO. The language selector also stores the visitor preference in `localStorage` under `chessview_locale` and mirrors it to a lightweight cookie for future server-side routing.

Human-facing routing is automatic:

- `/` redirects to the stored cookie language, or the browser `Accept-Language`, or English.
- `/events` and `/events/:id` redirect to the preferred localized route.
- The visible localized URL remains the canonical URL so search engines can index each language separately.

## Theme and Branding

The first brand direction is intentionally quiet and operational:

- Chess View wordmark with a compact `CV` mark.
- Light/dark themes with `chessview_theme` persisted in `localStorage`.
- Public pages use a restrained green/teal/accent palette to match tournament-management workflows.
- The current implementation uses plain CSS variables instead of Tailwind to stay close to the existing React frontend and keep the initial design system easy to audit.

## Slug Strategy

Event pages should use stable slugs generated by the backend.

- Canonical public event URL: `/:locale/events/:slug`.
- ID-based URLs still work for compatibility, but redirect to the slug when the API returns one.
- Sitemap entries use slugs when available.
- Country and source pages use normalized slugs derived from public event metadata.
- The sitemap is dynamic so newly imported scraper events can appear without rebuilding the app.
- If an event title changes, avoid changing the slug unless the organizer explicitly wants a URL change.

## Local Setup

```bash
npm install
cp .env.example .env.local
npm run dev -- --hostname 127.0.0.1 --port 3001
```

The app expects the backend at `http://127.0.0.1:5000/api` by default.

In development, API fetches use `cache: "no-store"` so scraper imports appear immediately. In production, API fetches use Next.js revalidation so public pages can be cached while still refreshing as tournament data changes.

## Environment Variables

```env
API_BASE_URL=http://127.0.0.1:5000/api
NEXT_PUBLIC_SITE_URL=http://127.0.0.1:3001
```

`API_BASE_URL` is server-only. Do not put secrets in variables prefixed with `NEXT_PUBLIC_`; those are exposed to browsers.

Production domain and production API URL are intentionally unset for now. Set them per deployment once the domain and API host are decided.

## Security and Open Source Notes

- `.env.local` is ignored and must not be committed.
- No private API keys are required for this public app.
- Security headers are configured in `next.config.mjs`.
- The package is marked private to avoid accidental npm publishing, not to make the repository closed source.

## Admin Frontend Split

This Next.js repository is only for the public, SEO-oriented site. Authenticated organizer/admin workflows stay in the React frontend repository unless the project later decides to migrate them.

## Backend

Run the backend from `chessview.org-backend`:

```bash
npm install
cp .env.example .env
npm run dev
```

For scraper work, use MongoDB persistence with `MEMORY_STORE=false`. For local demos without MongoDB, set `MEMORY_STORE=true` in the backend `.env`.

The public app automatically exposes imported tournaments through localized event pages, country pages, source pages, JSON-LD, robots, and the dynamic sitemap.
